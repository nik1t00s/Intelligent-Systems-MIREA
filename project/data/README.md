# Данные

В проекте используется реальный открытый датасет Jira Dataset:

```text
https://www.kaggle.com/datasets/cesaranasco/jira-dataset
```

Ожидаемый локальный путь:

```text
data/jira_dataset.csv
```

Файл не следует коммитить в публичный репозиторий без проверки условий Kaggle, поэтому в `.gitignore` добавлено правило для `data/*.csv`.

Минимальный сценарий получения данных:

1. Открыть страницу датасета на Kaggle.
2. Скачать CSV-файл.
3. Положить файл в `project/data/jira_dataset.csv`.
4. Запустить подготовку и обучение:

```bash
python -m src.train
```

Скрипт обучения сам сформирует подготовленную таблицу:

```text
artifacts/jira_issues_prepared.csv
```

Используемые исходные поля:

- `Issue key`
- `Issue Type`
- `Priority`
- `Created`
- `Resolved`
- `Component/s`
- `Summary`
- `Description`

Целевая переменная `is_delayed` строится по реальному времени закрытия issue.
## Демонстрационная выборка

Для проверки структуры данных в репозитории хранится небольшой файл:

```text
data/demo_sample.csv
```

Он содержит несколько обезличенных строк с финальными признаками модели и целевой переменной. Этот файл нужен только для демонстрации формата данных и не заменяет полный Jira Dataset.
